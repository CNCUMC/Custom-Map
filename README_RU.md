## Обзор

**Custom Map** позволяет загружать пользовательские карты в Casualties Unknown.

- **Многоуровневые карты** — Поддержка нескольких уровней с независимыми настройками, структурами и командами
- **Система функций** — Настройка мин, турелей, пружинных платформ, шипастых ловушек, звуковых пушек, гейзеров,
  медвежьих капканов и многого другого
- **Пользовательский экран загрузки** — Отображение прогресса в реальном времени
- **Интерфейс команд** — Полный набор команд `cm` для управления картами в игре
- **Многоязычность** — English, 简体中文, 繁體中文, Русский
- **На базе Custom Structures** — Расширение [Custom Structures](https://www.nexusmods.com/scavprototype/mods/9) с
  поддержкой уровней, системой функций и игровыми командами

## Установка

1. Установите [BepInEx 5.x](https://github.com/BepInEx/BepInEx) для Casualties Unknown.
2. Установите [CUCoreLib](https://github.com/jimmyking9999999/CUCoreLib) — поместите `CUCoreLib.dll` в
   `BepInEx/plugins/`.
3. Установите [Bark](https://github.com/CNCUMC/Bark) — поместите `Bark.dll` в `BepInEx/plugins/Bark/`.
4. Установите [Custom Structures](https://www.nexusmods.com/scavprototype/mods/9) — поместите в `BepInEx/plugins/`.
5. Установите Custom Map, поместите в `BepInEx/plugins/Custom Map`.
6. Разместите папки карт в `Maps/` (рядом с исполняемым файлом игры).

## Быстрый старт

### Загрузка встроенного шаблона

Custom Map включает встроенный шаблон карты. Включите **Start Game Use Map** в настройках мода, выберите шаблон, затем
начните новую игру.

### Использование пользовательских карт

1. Создайте папку карты в `Maps/имя-карты/` (см. [Структура карты](#структура-карты)).
2. Запустите игры. Карты появятся в выпадающем списке `Настройки мода → Custom Map → First Use Map`.
3. Введите `cm select имя-карты` в консоли или выберите из выпадающего списка перед началом.

## Команды

Все команды используют префикс `cm`.

| Команда                       | Описание                                           |
|-------------------------------|----------------------------------------------------|
| `cm help`                     | Показать справку                                   |
| `cm reload`                   | Перезагрузить текущую карту с диска                |
| `cm load`                     | Перезагрузить все карты из папки Maps              |
| `cm savereload`               | Сохранить и перезагрузить текущую карту            |
| `cm info`                     | Показать информацию о текущей карте                |
| `cm spawn`                    | Телепортация на точку спавна                       |
| `cm select <id>`              | Выбрать карту по ID, имени или индексу             |
| `cm list`                     | Список всех доступных карт                         |
| `cm feature [имя] [значение]` | Список или настройка функций карты                 |
| `cm waypoint [list\|get]`     | Управление путевыми точками                        |
| `cm save`                     | Сохранить текущую карту на диск                    |
| `cm layer [n]`                | Показать или переключить уровень                   |
| `cm exit <цель>`              | Выйти из карты — `none` (ванильная) или `tutorial` |

## Структура карты

Карта — это папка в `Maps/` со следующей структурой:

```
Maps/
└── имя-карты/
    ├── map.json                      # Метаданные карты
    ├── layers/
    │   ├── layer1/                   # Папка уровня 1
    │   │   ├── layer.json            # Свойства уровня
    │   │   ├── command.json          # Команды уровня (необязательно)
    │   │   └── *.m2.json             # Файлы структур (необязательно)
    │   └── layer2/                   # Папка уровня 2 (необязательно)
    │       ├── layer.json
    │       ├── command.json
    │       └── *.m2.json
    ├── command.json                  # Глобальные команды — применяются ко всем уровням как запасной вариант (необязательно)
    ├── feature/
    │   ├── world/
    │   │   ├── settings.json         # Настройки мира (гравитация, полная яркость, флаги пропуска)
    │   │   ├── mine.json             # Конфигурация мин (необязательно)
    │   │   ├── jump_pad.json         # Конфигурация пружинных платформ (необязательно)
    │   │   ├── turret.json           # Конфигурация турелей (необязательно)
    │   │   ├── sound_cannon.json     # Конфигурация звуковых пушек (необязательно)
    │   │   ├── spike_stabber.json    # Конфигурация шипастых ловушек (необязательно)
    │   │   ├── geyser.json           # Конфигурация гейзеров (необязательно)
    │   │   └── beartrap.json         # Конфигурация медвежьих капканов (необязательно)
    │   └── player/
    │       └── xp.json               # Конфигурация опыта (необязательно)
    └── lang/
        ├── EN.json                   # Английская локализация
        ├── zh-CN.json                # Локализация (упрощённый китайский)
        └── zh-TW.json                # Локализация (традиционный китайский)
```

### map.json

```json
{
  "id": "mymap",
  "version": "1.0.0",
  "author": [
    "ИмяАвтора"
  ]
}
```

| Поле      | Тип        | Описание                        |
|-----------|------------|---------------------------------|
| `id`      | `string`   | Уникальный идентификатор карты  |
| `version` | `string`   | Версия карты (например `1.0.0`) |
| `author`  | `string[]` | Список имён авторов             |

## Уровень

Каждый уровень — это папка в `layers/` (например `layers/layer1/`).

### layer.json

```json
{
  "type": "Debug",
  "spawn": [
    0,
    0
  ],
  "x": -68,
  "y": 62,
  "skip_terrain": true,
  "skip_structures": true,
  "skip_background": true,
  "waypoints": [],
  "items": []
}
```

| Поле              | Тип        | По умолч. | Описание                                                                             |
|-------------------|------------|-----------|--------------------------------------------------------------------------------------|
| `type`            | `string`   | `Debug`   | Тип сцены: `Debug`, `Wasteland`, `TemperateForest`, `RockDesert`, `Tutorial`, `None` |
| `spawn`           | `number[]` | `[0,0]`   | Позиция спавна `[x, y]`                                                              |
| `x`               | `int`      | `0`       | Смещение уровня по X в мировых координатах                                           |
| `y`               | `int`      | `0`       | Смещение уровня по Y в мировых координатах                                           |
| `skip_terrain`    | `bool`     | `true`    | Пропустить генерацию ванильного рельефа                                              |
| `skip_structures` | `bool`     | `true`    | Пропустить генерацию ванильных структур                                              |
| `skip_background` | `bool`     | `true`    | Пропустить генерацию ванильного фона                                                 |
| `waypoints`       | `array`    | `[]`      | Список путевых точек (для `cm waypoint`)                                             |
| `items`           | `array`    | `[]`      | Предметы, выдаваемые при спавне                                                      |

### Файлы структур (*.m2.json)

Каждый файл `.m2.json` в папке уровня определяет одно размещение структуры:

```json
{
  "structure": "ИмяСтруктуры",
  "x": 10,
  "y": 20
}
```

| Поле        | Тип      | Описание                                                       |
|-------------|----------|----------------------------------------------------------------|
| `structure` | `string` | Имя структуры (зарегистрированной через мод Custom Structures) |
| `x`         | `int`    | Координата X в мировых координатах                             |
| `y`         | `int`    | Координата Y в мировых координатах                             |

## Система команд

### Глобальные команды (`command.json` в корне карты)

Применяются ко **всем уровням** как запасной вариант. Если у уровня есть собственный `command.json`, приоритет имеет
уровень.

```json
{
  "once_commands": [
    "команда1",
    "команда2"
  ],
  "loop_commands": [
    "циклическая_команда1"
  ],
  "loop_interval": 10.0
}
```

### Команды уровня (`command.json` в папке уровня)

Переопределяют глобальные команды для конкретного уровня. Формат аналогичен выше.

| Поле            | Тип        | Описание                                         |
|-----------------|------------|--------------------------------------------------|
| `once_commands` | `string[]` | Команды, выполняемые один раз при загрузке карты |
| `loop_commands` | `string[]` | Команды, выполняемые циклически                  |
| `loop_interval` | `float`    | Интервал в секундах между циклическими командами |

## Функции

Каждый файл функции в `feature/world/` или `feature/player/` настраивает соответствующую игровую сущность.

| Функция              | Описание                                                                                 |
|----------------------|------------------------------------------------------------------------------------------|
| `settings.json`      | Настройки мира: `full_bright`, `gravity`, `jump_limit`, `climb_limit`, `forgiving_level` |
| `mine.json`          | Мины: перезарядка, неуязвимость                                                          |
| `jump_pad.json`      | Пружинные платформы: перезарядка, сила, без света                                        |
| `turret.json`        | Турели: перезарядка, мощность выстрела, дальность, неуязвимость, без света               |
| `sound_cannon.json`  | Звуковые пушки: перезарядка, дальность, время зарядки, неуязвимость                      |
| `spike_stabber.json` | Шипастые ловушки: урон, перезарядка, неуязвимость, без света, звук                       |
| `geyser.json`        | Гейзеры: перезарядка, длительность, дальность, время тряски, без жидкости                |
| `beartrap.json`      | Медвежьи капканы: урон, перезарядка, неуязвимость                                        |
| `xp.json`            | Опыт: уровни силы/упругости/интеллекта и диапазоны опыта                                 |

| Настройка                  | Описание                                       | По умолч.  |
|----------------------------|------------------------------------------------|------------|
| `More Logs`                | Подробное логирование                          | `false`    |
